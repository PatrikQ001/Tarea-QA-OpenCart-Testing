1. Plan de Pruebas Unitarias

Aseguramiento de calidad y estabilidad del núcleo del sistema

1.1 Objetivo

Asegurar la calidad y el correcto funcionamiento a nivel de código de los componentes individuales de OpenCart. Se validará que las funciones y métodos de los módulos seleccionados retornen los valores esperados ante diferentes escenarios (datos válidos, inválidos y casos extremos).

1.2 Alcance

El plan cubrirá exclusivamente la capa lógica (controladores y modelos) de los siguientes 6 módulos críticos del sistema:

  • Clientes y Autenticación: Lógica de validación de registro, verificación de formato de credenciales, recuperación de contraseñas y sanitización de perfiles.
  • Catálogo y Búsqueda: Lógica de ordenamiento de productos, algoritmos de filtrado por características, sanitización de la barra de búsqueda y validación de reglas para reseñas.
  • Carrito y Flujo de Checkout: Cálculos matemáticos del carrito (agregar/eliminar), sumatorias de subtotales, y aplicación lógica de cupones de descuento.
  • Ventas e Inventario: Funciones de validación de estados de pedido (Pendiente, Completado, Rechazado) y la lógica matemática para el descuento de stock.
  • Localización, Impuestos y Configuración: Lógica de conversión de múltiples monedas, aplicación de reglas por zonas geográficas y validación de formatos de configuración.

2.¿Qué es OpenCart?

OpenCart es un sistema de gestión de contenidos (CMS) de código abierto diseñado específicamente para el comercio electrónico. Proporciona una base profesional y fiable para construir una tienda online exitosa.

Está desarrollado con el lenguaje de programación PHP y bases de datos MySQL/MariaDB. Su arquitectura sigue el patrón de diseño MVC-L (Model-View-Controller-Language), una variante del MVC tradicional que separa de forma estricta la lógica de negocio, la interfaz de usuario y los archivos de traducción para la internacionalización.

Breve Reseña Histórica

OpenCart fue creada originalmente en 1998 por Christopher G. Mann como una aplicación de comercio electrónico mucho más sencilla. La primera versión pública fue lanzada el 11 de mayo de 1999 y estaba escrita en el lenguaje Perl. Esta primera versión fue abandonada y retomada en 2009 por el desarrollador Daniel Kerr al ver potencial en la aplicación decidió reescribir OpenCart por completo desde cero cambiando el lenguaje obsoleto de Perl por PHP.

OpenCart permite la capacidad de gestionar múltiples tiendas desde un único panel de administración, mercado de extensiones y pasarela de pagos integradas.

3. Justificación de Elección

La selección de OpenCart como el Sistema Bajo Prueba se fundamenta bajo el cumplimiento técnico de los criterios y exigencias de la rúbrica de evaluación:

Compatibilidad de Licencia

OpenCart está distribuido bajo la licencia GNU General Public License (GPLv3). Esta licencia de código abierto cumple con el requisito de permitir el forking, la modificación del código fuente y el uso libre sin ningún tipo de restricciones académicas o legales, permitiendo la inyección de las suites de pruebas sin inconvenientes.

Dominio Comercial Altamente Transaccional (PyMEs)

El dominio de la plataforma corresponde al e-commerce, enfocado directamente en soluciones empresariales para pequeñas y medianas empresas (PyMEs). Desde la perspectiva de QA, este dominio posee flujos de ejecución críticos como la gestión de sesiones, carrito de compras, cálculo automatizado de impuestos y pasarelas de pago.

Stack Tecnológico Vigente en la Industria

El producto utiliza un stack basado en PHP y MySQL/MariaDB, bajo el patrón arquitectónico MVC-L. OpenCart permite implementar pruebas unitarias (Unit Testing) efectivas sin la necesidad de levantar dependencias complejas de la interfaz gráfica, utilizando herramientas como PHPUnit.

Complejidad Arquitectónica y Volumetría Adecuada (KLOC)

OpenCart clasifica como un proyecto de complejidad mediana ideal para un entorno académico controlado. Cuenta con un rango de líneas de código fuente entre las 25 KLOC y 45 KLOC cumpliendo con el estándar requerido. Además posee de más de 5 módulos core claramente definidos:

  • Login y Registro
  • Carrito de Compras
  • Checkout / Pago
  • Catálogo y Búsqueda
  • Gestión de Inventario
  • Sistema de Reseñas

Infraestructura DevOps Adaptable (CI/CD)

El proyecto permite una integración nativa con herramientas como Docker y Docker Compose para empaquetar el servidor web (Apache/Nginx), el motor PHP y la base de datos MySQL.

Idoneidad para Pruebas de Regresión

Al ser un software maduro pero en constante actualización, OpenCart es idóneo para simular escenarios reales donde la introducción de nuevas extensiones o parches requiere un conjunto robusto de pruebas de regresión para asegurar que las características núcleo no se corrompan a lo largo de los diferentes Sprints.
